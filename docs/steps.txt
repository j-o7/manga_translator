Here’s a **detailed, step-by-step checklist for the entire COMET project**, end-to-end, including what to save at each stage so your milestone + final report are easy.

---

# 0) Repo + folders (do this once)

1. Create repo root `COMET/`
2. Create folders:

   * `data/raw/` (any image types)
   * `data/pages_png/` (converted PNG pages)
   * `outputs/`

     * `bubbles_overlay/`
     * `bubble_crops/`
     * `bubble_masks/`
     * `ocr_json/`
     * `translated_json/`
     * `inpainted_bubbles/`
     * `final_pages/`
     * `debug/`
   * `scripts/`
3. Add:

   * `requirements.txt`
   * `README.md`

---

# 1) Data ingestion + conversion

4. Put pages into `data/raw/`
5. Convert everything to PNG:

   * `scripts/convert_to_png.py` or `.sh`
6. Verify `data/pages_png/` has all pages and they open correctly
7. Pick a **small dev subset** (5–10 pages) to iterate fast

Artifacts to save:

* `data/pages_png/*.png`

---

# 2) Speech bubble detection (page → bubble boxes)

8. For each page image:

   * Load page
   * Convert to grayscale
   * Denoise (Gaussian blur)
   * Edge detect (Canny)
   * Morphological close to connect edges
   * Find contours
   * Compute bounding boxes
9. Filter candidate boxes:

   * Min/max area ratio
   * Min width/height
   * Aspect ratio constraints
   * Optional “whiteness inside box” heuristic
10. (Optional but helpful) Deduplicate:

* Remove boxes heavily contained in larger box

11. Save:

* Page overlay image with numbered boxes
* Individual bubble crops per page

Artifacts to save:

* `outputs/bubbles_overlay/page_XX.png`
* `outputs/bubble_crops/page_XX/bubble_###.png`
* Optional: `outputs/debug/edges_page_XX.png`

---

# 3) Bubble interior segmentation (bubble crop → writable mask)

Goal: get a **mask of the interior** (writable area), not the outline.

12. For each bubble crop:

* Convert to grayscale
* Threshold (adaptive or Otsu) for white-ish region
* Remove border outline:

  * Erode then dilate (opening) OR
  * Use flood fill from center to capture interior

13. Post-process mask:

* Remove tiny components
* Fill small holes

14. Save mask as PNG (white=inside, black=outside)

Artifacts:

* `outputs/bubble_masks/page_XX/bubble_###_mask.png`
* Optional debug: masked overlay

---

# 4) OCR (bubble crop + mask → extracted text)

15. For each bubble:

* Apply mask (OCR only inside writable area)
* Run OCR engine (Tesseract / PaddleOCR / Google Vision)

16. Extract:

* Text string
* Confidence (if available)
* Bounding boxes of text lines (if available)
* Orientation (horizontal vs vertical) if supported

17. Save results per page as JSON

Artifacts:

* `outputs/ocr_json/page_XX.json`
* Optional: `outputs/debug/ocr_overlay_page_XX.png` (OCR boxes drawn)

Metrics to compute:

* Average OCR confidence
* OCR failure rate (# empty outputs)
* Average characters per bubble

---

# 5) Translation (bubble text → English)

18. For each bubble’s extracted text:

* Clean text (remove weird OCR artifacts)
* Translate using:

  * MarianMT / NLLB / Google Translate API

19. Save translation JSON with:

* Original text
* Translated text
* Length expansion ratio

Artifacts:

* `outputs/translated_json/page_XX.json`

Metrics:

* Avg length expansion ratio
* % bubbles where expansion causes overflow later

---

# 6) Text removal (bubble crop + mask → clean bubble)

20. For each bubble crop:

* Remove original text using ONE method:

  * Fast baseline: fill mask region with white (works surprisingly often)
  * Better: OpenCV inpainting (`cv2.inpaint`)

21. Save cleaned bubble image

Artifacts:

* `outputs/inpainted_bubbles/page_XX/bubble_###.png`

---

# 7) Layout-aware text fitting (core contribution)

Goal: choose font size + line breaks to fit inside the mask.

22. Inputs:

* Clean bubble image
* Mask of writable region
* Translated text

23. Implement fitting loop:
24. Start with large font size
25. Wrap text into lines (greedy word wrap)
26. Render text to a temporary canvas
27. Check fit:

    * All text pixels must fall inside mask
    * No overflow beyond bubble bounding box
28. Reduce font size and retry
29. When fit succeeds:

* Compute and log:

  * Final font size
  * Number of lines
  * Fit success/failure

25. Save:

* Rendered bubble with English text

Artifacts:

* `outputs/debug/fitted_text/page_XX/bubble_###.png`
* Or directly into `outputs/final_pages/` later

Metrics:

* Overflow rate
* Avg final font size
* Avg lines per bubble

---

# 8) Compositing (rendered bubble → final translated page)

26. For each bubble:

* Take rendered bubble region
* Paste back into the original page at its bbox location

27. Save final page

Artifacts:

* `outputs/final_pages/page_XX.png`

Also save side-by-side visuals for report:

* `outputs/debug/comparison_page_XX.png` (before/after)

---

# 9) Evaluation (quant + qual)

### Quantitative

28. Bubble detection:

* bubbles detected per page
* (optional) precision/recall on 2–3 hand-labeled pages

29. OCR:

* avg confidence
* empty extraction rate

30. Translation:

* expansion ratio stats
* small manual rating on 20 bubbles (adequate/incorrect)

31. Layout:

* overflow rate
* font size stats

### Qualitative

32. Provide 3–5 before/after pages
33. Provide failure case gallery:

* vertical text
* stylized fonts
* dense bubbles
* missed detections

34. Explain error propagation:

* missed bubble → no translation
* bad OCR → wrong translation
* expansion → overflow

---

# 10) Milestone deliverables (2 pages)

What you MUST have for milestone:
35. System diagram (pipeline)
36. At least:

* bubble detection working + overlays
* OCR + translation on bubbles
* 1 fully re-typeset page (even if crude)

37. Table with early metrics
38. Failure analysis + next steps

---

# 11) Final report deliverables (6 pages)

For the final report, add:
39. Improved segmentation (better masks)
40. Improved inpainting (cleaner text removal)
41. Stronger text fitting (mask-aware, not just bbox-aware)
42. More pages (20–50)
43. More thorough evaluation and ablations:

* OCR engine comparison
* detection method comparison
* inpainting method comparison

44. Clear discussion of limitations + future work

---

## If you want the “exact order to implement”

Implement in this strict order:

1. Convert → PNG
2. Bubble detection + overlays + crops
3. OCR on crops
4. Translate text
5. Quick text removal (white fill)
6. Simple bbox-based text fitting
7. Composite final pages
8. Metrics + figures
9. Upgrade to mask-based fitting + inpainting (final report)
